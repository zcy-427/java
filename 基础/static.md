# static关键字

核心作用是**将成员（属性、方法、代码块、内部类）标记为「类级别」**，而非「实例级别」—— 即成员属于整个类，不依赖具体对象实例，无需创建对象即可访问，且所有对象共享该成员。

---

### 静态属性

所有对象共享，仅类加载时初始化一次，可以直接通过类名访问

### 静态方法

**属于类而非对象**，无需创建对象，可以直接通过类名访问，静态方法不能够重写

### 静态代码块

静态代码块是「用 `static {}` 包裹的代码片段」，**仅在类加载时执行一次**（早于构造器），优先级高于对象的构造方法；核心用途是初始化静态属性（如复杂逻辑赋值、加载配置文件、注册驱动）。

### 静态导入

通过 `import static` 语法，可直接导入类的静态属性或静态方法，调用时无需写「类名。成员名」，简化代码（但需避免滥用，否则降低可读性）。

语法：

```java
// 导入单个静态成员
import static 包名.类名.静态成员名;

// 导入类中所有静态成员（慎用，可能导致命名冲突）
import static 包名.类名.*;
```

### 关键注意

1.`static`方法**不能够重写**

2.静态内部类不持有外部类引用，无需外部类实例即可创建

3.静态属性**非**线程安全，需通过 `synchronized` 或 `Atomic` 原子类保证线程安全。
